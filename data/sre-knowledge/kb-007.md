# OrderService order status update failure

## Symptoms
Updating order status fails on the OrderService PUT /api/orders/{id}/status endpoint. HTTP 404 Not Found or HTTP 400 Bad Request is returned.

## Possible Causes
1. The specified order ID does not exist
2. The status transition is invalid (for example: a reverse transition from Delivered → Pending)
3. The Cosmos DB partition key (UserId) does not match
4. The order has been canceled (transitions from the Cancelled status are not allowed)

## Verification Steps
1. Check the current order status with GET /api/orders/{id}
2. Verify that the status transition is valid: Pending → Confirmed → Processing → Shipped → Delivered
3. Check the relevant document directly in the Cosmos DB 'orders' container

## Mitigation
- Resend the request using the correct order ID
- Update the status according to the status transition rules
- Recreate canceled orders as new orders

## Valid Status Transitions
- Pending → Confirmed, Cancelled
- Confirmed → Processing, Cancelled
- Processing → Shipped
- Shipped → Delivered
- Delivered, Cancelled → (no transitions allowed)
