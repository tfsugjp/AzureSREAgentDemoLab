# System architecture overview

## GlobalAzureDemo2026 Architecture

### Service Composition
This system consists of three microservices:

1. **CatalogService** (port 5001)
   - Product catalog management (CRUD)
   - Category management
   - Inventory management
   - Product search
   - Cosmos DB containers: categories, products, inventory

2. **OrderService** (port 5002)
   - Order management
   - Order status tracking
   - Order cancellation
   - Cosmos DB container: orders

3. **NotificationService** (port 5003)
   - Notification delivery
   - Read/unread management
   - Cosmos DB container: notifications

### Shared Components (SharedLibrary)
- Microsoft Entra ID JWT authentication
- Cosmos DB connection management
- OpenTelemetry instrumentation (traces, metrics, logs)
- Serilog structured logging
- Common model definitions

### Infrastructure
- **Compute**: Azure Container Apps (or AKS + AGC)
- **Database**: Azure Cosmos DB (NoSQL API, Serverless)
- **Authentication**: Microsoft Entra ID
- **Monitoring**: Application Insights + Log Analytics Workspace
- **Container registry**: Azure Container Registry
- **Search**: Azure AI Search
- **IaC**: Bicep (infra/main.bicep)
- **Deployment**: Azure Developer CLI (azd)
