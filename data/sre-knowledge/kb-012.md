# CatalogService API endpoint list

## CatalogService API Endpoints

### Product Management
- GET /api/products - Retrieve all products
- GET /api/products/{id} - Retrieve product details
- POST /api/products - Create product
- PUT /api/products/{id} - Update product
- DELETE /api/products/{id} - Delete product

### Product Search
- GET /api/products/search?q={query} - Search products
  - Note: q=premium has a known performance issue (see kb-001)
  - Note: HTTP 500 occurs when q is longer than 100 characters (see kb-002)

### Category Management
- GET /api/categories - Retrieve all categories
- GET /api/categories/{id} - Retrieve category details

### Inventory Management
- GET /api/inventory/{productId} - Retrieve inventory information
- PUT /api/inventory/{productId} - Update inventory information

### Health Checks
- GET /health - Liveness check
- GET /health/ready - Readiness check

### Data Models
- Product: id, name, description, price, categoryId, categoryName, tags, imageUrl, isActive
- Category: id, name, description
- InventoryItem: id, productId, quantity, reservedQuantity, reorderThreshold

### Seed Data
- 5 categories: Electronics, Clothing, Food, Books, Premium
- 20 products (4 products per category)
- 20 inventory items
