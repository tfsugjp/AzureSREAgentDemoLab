# Troubleshooting Entra ID authentication errors

## Symptoms
Requests to API endpoints return HTTP 401 Unauthorized.

## Possible Causes
1. The JWT token is missing or invalid
2. The Entra ID application registration is not configured correctly
3. Tenant ID, client ID, or Audience settings do not match
4. The token has expired
5. The disableAuthentication flag is false (default), and access is attempted without a token

## Verification Steps
1. Verify that the request header includes Authorization: Bearer <token>
2. Check the status of the Entra ID application registration in the Azure Portal
3. Check the values of the ENTRA_TENANT_ID, ENTRA_CLIENT_ID, and ENTRA_AUDIENCE environment variables
4. Decode the token (jwt.ms) and verify the iss and aud claims

## Mitigation
- Disable authentication for workshops: azd env set DISABLE_AUTHENTICATION true
- Recreate the app registration: Run scripts/setup-entra-app.ps1
- Acquire a new token: Use the az account get-access-token --resource <audience> command
