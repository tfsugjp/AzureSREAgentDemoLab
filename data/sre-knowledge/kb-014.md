# NotificationService API endpoint list

## NotificationService API Endpoints

### Notification Management
- GET /api/notifications - Retrieve all notifications
- GET /api/notifications?userId={userId} - Retrieve notifications by user
- GET /api/notifications/{id} - Retrieve notification details
- POST /api/notifications - Create notification
- PUT /api/notifications/{id}/read - Mark as read
- DELETE /api/notifications/{id} - Delete notification

### Health Checks
- GET /health - Liveness check
- GET /health/ready - Readiness check

### Data Models
- Notification: id, userId, title, message, type, isRead, relatedEntityId, createdAt
- NotificationType: OrderConfirmation, ShipmentUpdate, Promotion, Info, Alert

### Seed Data
- 15 notifications (for 3 users)
- Includes various types: order confirmations, shipment updates, promotions, information, and alerts
