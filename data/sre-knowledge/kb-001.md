# CatalogService product search response latency (Premium search path)

## Symptoms
When requests are sent to the CatalogService product search endpoint GET /api/products/search?q=premium, responses take several seconds or longer. Normal search queries return within 100ms, but only queries containing premium show significant latency.

## Root Cause
In the search logic in ProductSearchService.cs, when the query string contains "premium", a dedicated search path is executed. This search path has the following issues:
1. It uses an anti-pattern that retrieves all products from Cosmos DB in a single batch (SELECT * FROM c WHERE c.isActive = true)
2. It calls Thread.Sleep(100) for each retrieved product, synchronously blocking threads
3. Because it uses synchronous processing instead of the async/await pattern, it causes thread pool starvation

## Impact
- The CatalogService /api/products/search endpoint
- Only queries containing premium are affected
- Under high load, thread pool starvation can also impact other requests

## Mitigation
1. Immediate action: Configure rate limiting to restrict traffic for premium searches
2. Permanent fix: Replace Thread.Sleep(100) with await Task.Delay(100), or remove the simulated external service call
3. Permanent fix: Stop retrieving all products in a single batch and filter directly in the Cosmos DB query

## Verification Commands
```bash
curl -w '\nTotal: %{time_total}s\n' http://localhost:5001/api/products/search?q=premium
curl -w '\nTotal: %{time_total}s\n' http://localhost:5001/api/products/search?q=wireless
```
