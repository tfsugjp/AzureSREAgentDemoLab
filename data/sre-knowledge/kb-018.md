# ACR (Azure Container Registry) image pull failures

## Symptoms
Container Apps cannot pull container images from ACR. The Container Apps revision enters the Failed state.

## Possible Causes
1. The ACR Pull role assignment is not configured correctly
2. The Container Apps managed identity is disabled
3. The ACR admin user is disabled (by design, adminUserEnabled: false)
4. The image tag does not exist
5. Access is blocked by ACR network settings

## Verification Steps
1. Check ACR Access control (IAM) in the Azure Portal
2. Verify that the Container Apps system-managed identity is enabled
3. Check whether the image exists in the ACR repository
4. Check the Container Apps system logs for image pull error details

## Mitigation
- Verify role assignments: Confirm that the catalogAcrPull, orderAcrPull, and notificationAcrPull resources in main.bicep were created correctly
- Managed identity: Confirm that systemAssigned is enabled in the identity setting in container-app.bicep
- Push the image again: Push the latest image to ACR with azd deploy
- ACR network: Confirm that publicNetworkAccess is set to 'Enabled'

## Related Configuration
- Bicep: containerRegistry resource and each AcrPull role assignment in infra/main.bicep
- Bicep: registryServer parameter in infra/modules/container-app.bicep
