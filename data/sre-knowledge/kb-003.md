# Troubleshooting Cosmos DB connection errors

## Symptoms
Cosmos DB connection errors occur when each microservice (CatalogService, OrderService, NotificationService) starts up or processes requests. The error message includes Microsoft.Azure.Cosmos.CosmosException.

## Possible Causes
1. The Cosmos DB account connection string is not configured correctly
2. Network connectivity issues (firewall, VNet configuration)
3. The Cosmos DB account does not exist or has been deleted
4. RU limits are being reached in Serverless capacity mode
5. The Cosmos DB database or containers have not been created

## Verification Steps
1. Verify that the CosmosDb__ConnectionString environment variable is correct
2. Check the Cosmos DB account status in the Azure Portal
3. Check whether public access is enabled in the network settings
4. Verify that the 'GlobalAzureDemo' database and each container (categories, products, inventory, orders, notifications) exist

## Mitigation
- Correct the connection string: azd env set CosmosDb__ConnectionString '<correct connection string>'
- If containers do not exist: Restart the services because each service's StartupInitializationService creates containers automatically
- RU limits: Check Cosmos DB metrics for 429 errors and increase capacity if necessary

## Related Configuration
- Bicep: cosmosAccount resource in infra/main.bicep
- App settings: CosmosDb__ConnectionString, CosmosDb__DatabaseName environment variables
