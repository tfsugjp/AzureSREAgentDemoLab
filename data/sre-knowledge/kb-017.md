# Troubleshooting Docker builds and local development environments

## Symptoms
Building Docker containers or starting them with docker-compose fails.

## Common Errors and Mitigations

### Build error: NuGet package restore failure
Cause: Network connectivity issue or authentication error for a private NuGet feed
Mitigation: Rebuild with docker build --no-cache and check network connectivity

### Build error: Incorrect Dockerfile context
Cause: Each service's Dockerfile uses the root directory as the context
Mitigation: Build from the root, for example docker build -f src/CatalogService/Dockerfile -t catalog .

### Startup error: Port conflict
Cause: Local ports 5001, 5002, and 5003 are being used by other processes
Mitigation: Check port usage with netstat -tlnp | grep '500[1-3]' and stop the conflicting process

### Startup error: Cosmos DB connection failure
Cause: The local environment does not have a Cosmos DB emulator or connection string configured
Mitigation: Start Azure Cosmos DB Emulator, or set the connection string for Cosmos DB on Azure in an environment variable

## Local Development Commands
```bash
# ソリューション全体のビルド
dotnet build AzureSREAgentDemoLab.slnx

# Docker Compose でのローカル起動
docker-compose up --build

# 個別サービスの起動
cd src/CatalogService && dotnet run
```
