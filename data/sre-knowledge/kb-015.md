# Collecting and analyzing Container Apps logs

## Overview
Logs from each service are sent to Azure Monitor (Log Analytics Workspace) through the Container Apps Environment.

## How to Check Logs

### Azure Portal
1. Open the relevant Container Apps resource in the Azure Portal
2. Select [Monitoring] → [Logs]
3. Use KQL queries to search logs

### KQL Query Examples
```kql
// CatalogService のエラーログ
ContainerAppConsoleLogs_CL
| where ContainerAppName_s startswith 'ca-cat'
| where Log_s contains 'Error' or Log_s contains 'Exception'
| order by TimeGenerated desc
| take 50

// 特定の時間帯のリクエストログ
ContainerAppConsoleLogs_CL
| where TimeGenerated > ago(1h)
| where Log_s contains 'Searching products'
| order by TimeGenerated desc

// Container Apps のシステムログ
ContainerAppSystemLogs_CL
| where ContainerAppName_s startswith 'ca-'
| where Type_s == 'Error'
| order by TimeGenerated desc
| take 20
```

### Application Insights
The following telemetry data is available in Application Insights:
- Requests: Request information for each API endpoint
- Dependencies: Call information for Cosmos DB
- Exceptions: Stack traces for unhandled exceptions
- Traces: Structured logs from Serilog

## Serilog Structured Logs
Each service outputs structured logs using Serilog. Key log properties:
- RequestPath: Request path
- StatusCode: HTTP response code
- ElapsedMilliseconds: Request processing time
- UserId: Authenticated user ID
