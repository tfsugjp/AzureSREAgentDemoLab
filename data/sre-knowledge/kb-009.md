# Troubleshooting azd up deployment failures

## Symptoms
The azd up command fails. An error occurs during the provisioning phase or deployment phase.

## Common Errors in the Provisioning Phase
1. Resource providers not registered: Providers such as Microsoft.App and Microsoft.DocumentDB are not registered
2. Resource name conflicts: Duplicate globally unique names (Cosmos DB, ACR)
3. Quota limits: Regional resource quota exceeded
4. Insufficient Entra ID configuration: ENTRA_TENANT_ID, ENTRA_CLIENT_ID, and ENTRA_AUDIENCE are not set

## Common Errors in the Deployment Phase
1. Docker build error: Incorrect Dockerfile build context
2. ACR push error: Authentication error or network timeout
3. Container Apps deployment error: Image reference mismatch

## Verification Steps
1. Check current environment variables with azd env get-values
2. Check the resource group's deployment history in the Azure Portal
3. Check unregistered providers with az provider list --query "[?registrationState=='NotRegistered']"

## Mitigation
- Register resource provider: az provider register --namespace Microsoft.App
- Set environment variables: azd env set <KEY> <VALUE>
- Entra ID configuration: Run scripts/setup-entra-app.ps1
- Change region: azd env set AZURE_LOCATION <another region>
- Cleanup: Delete existing resources with azd down, then redeploy with azd up

## Prerequisites
- Azure CLI is installed
- Azure Developer CLI (azd) is installed
- Docker Desktop is running
- .NET 10 SDK is installed
