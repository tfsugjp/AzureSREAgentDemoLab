# OpenTelemetry tracing data is not displayed

## Symptoms
Tracing data is not displayed in Application Insights or a custom OTLP endpoint. Distributed traces are incomplete, or calls between services cannot be tracked.

## Possible Causes
1. The Application Insights connection string is not configured correctly
2. OpenTelemetry SDK initialization error
3. Container Apps Environment OpenTelemetry configuration is incorrect
4. Network connectivity issues to the OTLP endpoint
5. Some traces are missing due to the sampling rate setting (SamplingPercentage)

## Verification Steps
1. Check the connection string for the Application Insights resource
2. Check the Container Apps Environment openTelemetryConfiguration
3. Check each service's logs for OpenTelemetry initialization messages
4. Check real-time data in Application Insights Live Metrics

## Mitigation
- Verify Application Insights settings: Copy the connection string from the resource properties in the Azure Portal
- Container Apps settings: Verify that the destinations for logsConfiguration and tracesConfiguration are set to 'appInsights'
- Sampling rate: Set SamplingPercentage to 100 to collect all traces (default setting)
- Custom OTLP: Set the correct endpoint URL in the openTelemetryEndpoint parameter

## Related Configuration
- Bicep: applicationInsights resource and containerAppsEnvironment openTelemetryConfiguration in infra/main.bicep
- App: OpenTelemetry configuration in SharedLibrary
